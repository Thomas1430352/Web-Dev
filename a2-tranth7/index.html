Frontend Design

The idea of front end design should be thought of as designing a user interface with the best experience and usability for the end user. This is to mean that the developer will use a good visual design and a practical design. Where visual could be things like matching font, color scheme, and themes. A practical design can mean making it easy to navigate or intuitive in the buttons and iterfaces so that the application's usability increases. One way a developer can interpret usability of a website or application is by using the five "E"s known as effective, efficient, easy-to-navigate, error-free, and enjoyability.

Effective
Determined by how well the application or design helps to accomplish the user's goal's. The design should do what it is intended to do and should display accurate results.
Efficient
Determined by if the design uses the least amount of steps to reach end-goal or solution. The application should have the least amount of unneccesary steps to accomplish task at hand, and the results should be showed promptly.
Easy-to-Navigate
Determined by how practical and intuitive the design is. The user should understand how to use the application whether it's their first time or next time using it
Error-Free
Determined by how many problems occur for user when using applications. The application should be accessible and availble to the user, and bugs should not appear when the application is being used for it's intended purpose.
Enjoyable
Determined by how much the intended audiance enjoys the product. The application should be engaging to the user and should make the user want to come back to use it. Enjoyable can be determined by the other four "E"s, and also by design as well.
