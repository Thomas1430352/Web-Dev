Frontend Design

The idea of front-end design should be thought of as designing a user interface with the best experience and usability for the end user. This is to mean that the developer will use a good visual design and a practical design. Where visual could be things like matching font, color scheme, and themes. A practical design can mean making it easy to navigate or intuitive in the buttons and interfaces so that the application's usability increases. One way a developer can interpret usability of a website or application is by using the five "E"s known as effective, efficient, easy-to-navigate, error-free, and enjoyable.

Effective
Determined by how well the application or design helps to accomplish the user's goals. The design should do what it is created to do and should display accurate results.
Efficient
Determined by if the design uses the least amount of steps to reach the end goal or solution. The application should have the least amount of unnecessary steps to accomplish the task at hand, and the results should be shown promptly.
Easy-to-Navigate
Determined by how practical and intuitive the design is. The user should understand how to use the application whether it's their first time or next time using it
Error-Free
Determined by how many problems occur for the user when using applications. The application should be accessible and available to the user, and bugs should not appear when the application is being used for its intended purpose.
Enjoyable
Determined by how much the intended audience enjoys the product. The application should be engaging to the user and should make the user want to come back to use it. Enjoyable can be determined by the other four "E"s, and also by design as well.

Then there are page layout tags, which are a type of block element in HTML used to break apart web pages or to section off different content. The layout tags consist of these elements: header, nav, main, section, article, aside, figure, blockquote, footer, and div. The header tag is usually the banner or headline of the application, and the same across pages, where it includes things like name, publisher, logo, and slogan. The nav element is a way for users to navigate from different pages and can have multiple ways to navigate the application like search bars, menus, locations, or even links. The main element is just to section off the primary area where the content will be held, like your articles and sections. The section element is used to thematically group articles or contents that relate but aren't the same. Like a math book, has chapters with sub-chapters. The article section is typically going to be placed inside section tags, which are used to identify individual specific topics, and are usually headlined with a "h2". Aside elements usually have correlated information to the article and sit on the side. The figure and figcaption element are typically used like the aside element but have media files along with "figcaption" to accompany the media. The blockquote element is for quotations that are too long for a paragraph and are usually visually different from the paragraph. The footer is usually below the tag and holds all information related to legal information, and contact information, and can have links to other related content or passages. The footer should also contain websites owner's copyright statement, year of publication, and the owner's name. The div element is used to divide content when it does not seemingly fit in one of the other elements and usually has no style or dimensions.
